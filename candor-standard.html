01 — THE PROBLEM

Why this standard exists

Digital platforms today suppress content based on proprietary, opaque algorithms. Users have no way to know why a post was demoted, what criteria were used, whether the flagging was accurate, or what the platform's conflict of interest was.

This creates an environment where legitimate journalism, whistleblower disclosures, and community accountability content gets silenced alongside actual disinformation — with no transparency and no recourse.

The CANDOR Score Standard replaces suppression with scored transparency.

02 — CORE PRINCIPLE

Label, never suppress

"Content that scores low on the CANDOR scale receives a context label and a visible score. It is never hidden, shadow-banned, or algorithmically demoted without disclosure. Users see the score and decide for themselves."

This is the non-negotiable foundation of the standard. Any implementation that uses CANDOR scores to hide or algorithmically bury content — without user-visible disclosure — is not compliant.

04 — SCORE COMPONENTS

How the score is calculated

A CANDOR Score is computed from five weighted components. Platforms may adjust weights within the defined ranges.

ComponentDescriptionDefaultRange
Source CredibilityTrack record of the author or outlet0.300.20 – 0.50
Claim VerifiabilityCan claims be checked against primary sources?0.250.15 – 0.40
Public InterestDoes this serve the public good?0.200.10 – 0.30
Community VerificationHas the community confirmed or disputed this?0.150.05 – 0.25
Engagement AuthenticityAre engagements organic or manipulated?0.100.05 – 0.20

Penalties and bonuses are applied after component scoring. A full penalty/bonus table is available in the full specification.

05 — DISPLAY REFERENCE

The CANDOR Ring — SVG implementation

The reference implementation uses a circular progress ring. The ring fills proportionally to the score and renders in the score's color.

<svg width="52" height="52" viewBox="0 0 52 52">
  <!-- Background track -->
  <circle cx="26" cy="26" r="22" fill="none" stroke="#1E2535" stroke-width="3"/>
  <!-- Score ring — green, score=97 -->
  <circle cx="26" cy="26" r="22" fill="none" stroke="#1D9E75" stroke-width="3"
    stroke-dasharray="138.2" stroke-dashoffset="4.1"
    stroke-linecap="round" transform="rotate(-90 26 26)"/>
  <!-- Score number -->
  <text x="26" y="22" text-anchor="middle" fill="#1D9E75"
    font-size="13" font-weight="500" font-family="monospace">97</text>
  <!-- Label -->
  <text x="26" y="33" text-anchor="middle" fill="#1D9E75"
    font-size="6.5" font-family="monospace" letter-spacing="0.5">CANDOR</text>
</svg>

Calculating stroke-dashoffset for any score:

const radius = 22;
const circumference = 2 * Math.PI * radius; // 138.23
const offset = circumference - (score / 100) * circumference;

// Examples:
// score=97  → offset = 4.15   (nearly full ring, green)
// score=60  → offset = 55.29  (60% full ring, yellow)
// score=20  → offset = 110.58 (20% full ring, red)
06 — CONTEXT LABELS

Labels replace suppression

When a post scores below 75, a context label accompanies the score. Labels are informational only — they never restrict visibility.

08 — IMPLEMENTATION CHECKLIST

Are you compliant?

Use this checklist to verify your implementation meets the standard:

  • Scores display as integers 0–100
  • Green (#1D9E75) used for scores 75–100
  • Yellow (#EF9F27) used for scores 45–74
  • Red (#E24B4A) used for scores 0–44
  • The word "CANDOR" appears adjacent to every score
  • Verdict label displayed with every score
  • Context labels are informational only — no content hidden
  • Suppression events logged and publicly accessible
  • Restoration badge displayed on restored content
  • Score audit trail accessible via public link
  • IPFS verdict storage implemented or planned
  • Weights publicly documented
  • Governance participation enabled or planned
