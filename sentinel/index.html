Candor Sentinel API

Real-time bot detection and coordinated inauthentic behavior (CIB) protection for open social platforms, forums, and content pipelines. Send a request context, receive a threat score and recommended action in milliseconds.

https://sentinel.candor.network/v1
Beta access: Sentinel API is in closed beta. Contact api@candor.network to request access or start a trial.

Authentication

All API requests require your secret API key passed as a Bearer token. Keys are prefixed sk_sentinel_ and issued at account creation.

# Pass your key in the Authorization header
curl https://sentinel.candor.network/v1/evaluate \
  -H "Authorization: Bearer sk_sentinel_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"ip":"203.0.113.42","userAgent":"Mozilla/5.0..."}'
Keep keys secret. Never expose sk_sentinel_ keys in client-side code, public repos, or logs. Use environment variables.

Alternatively: X-API-Key header

curl https://sentinel.candor.network/v1/evaluate \
  -H "X-API-Key: sk_sentinel_your_key_here"

Quickstart

Evaluate a request in under 60 seconds. Pass the visitor's IP and user agent — Sentinel returns a score and recommended action.

// Node.js example
const response = await fetch('https://sentinel.candor.network/v1/evaluate', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.SENTINEL_API_KEY}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    ip:          req.ip,
    userAgent:   req.headers['user-agent'],
    headers:     req.headers,
    accountId:   req.user?.id,
    requestType: 'post',
    content:     req.body.text,
  }),
});

const { score, risk, action, signals } = await response.json();

if (action === 'block')     return res.status(429).json({ error: 'Blocked' });
if (action === 'challenge') return serveCaptcha(res);
if (action === 'throttle')  await sleep(1000);
Sentinel fails open — if the API is unreachable, treat the response as action: "allow" so legitimate traffic is never blocked by an outage.

Evaluate a request

The core endpoint. Send request context for a single visitor interaction and receive a threat assessment.

POST /v1/evaluate

Evaluates a single request context across 6 detection layers: network reputation, behavioral velocity, timing regularity, content analysis, account credibility, and CIB graph detection.

Request body

FieldTypeDescription
ip requiredstringIPv4 or IPv6 address of the visitor.
userAgent optionalstringHTTP User-Agent header. Strongly recommended — missing UA is itself a signal.
headers optionalobjectHTTP request headers. Used for browser fingerprinting. Sentinel sanitizes these — never sends sensitive headers downstream.
accountId optionalstringYour platform's user ID. Enables account credibility and CIB graph scoring.
content optionalstringPost or message text. Enables content analysis layer (entropy, URL density, crypto patterns). Pro+ only.
requestType optionalstringOne of request, post, login, register. Defaults to request.
timestamp optionalnumberUnix milliseconds. Defaults to current server time.

Response

scorenumberComposite threat score, 0–100. Higher = more likely bot or malicious actor.
riskstringLOW · MEDIUM · HIGH · CRITICAL
actionstringRecommended action: allow · challenge · throttle · block
signalsarrayDetected threat signals with layer, name, weight, and optional detail. See Signal Reference.
topSignalstringHighest-weight signal name. Useful for logging and audit trails.
fingerprintstring16-char hex request fingerprint (IP + UA + key headers). Stable across retries from same browser.
timestampnumberUnix ms when the evaluation completed.
usageobjectCurrent billing period usage: used, limit, remaining, overage.

Batch evaluate

Evaluate up to 100 request contexts in a single API call. Each item in the batch counts as one evaluation against your monthly limit. Pro plan and above.

POST /v1/evaluate/batch Pro+

Request body

FieldTypeDescription
requests requiredarrayArray of up to 100 request context objects. Each has the same shape as /v1/evaluate.

Example

{
  "requests": [
    { "ip": "203.0.113.1", "userAgent": "Mozilla/5.0...", "requestType": "post" },
    { "ip": "203.0.113.2", "userAgent": "python-requests/2.28" },
    { "ip": "203.0.113.3", "accountId": "user_abc123", "content": "Send ETH to 0x..." }
  ]
}

Response

completednumberNumber of requests successfully evaluated.
totalnumberTotal requests submitted.
resultsarrayAssessment for each request, in order. Each item includes index plus all standard evaluate fields.

API key management

POST /v1/account/keys

Generate a new API key. Key count is limited by your plan (Starter: 1, Pro: 5, Enterprise: 20).

FieldTypeDescription
label optionalstringHuman-readable label, e.g. "Staging" or "Worker-2".
The full key is only returned once at creation. Store it immediately in a secrets manager.
DELETE /v1/account/keys/:keyPrefix

Revoke an API key by its first 20 characters. Revoked keys are rejected immediately.

curl -X DELETE \
  https://sentinel.candor.network/v1/account/keys/sk_sentinel_a1b2c3d4e5 \
  -H "Authorization: Bearer sk_sentinel_your_key"

Response schema

Risk levels

Every evaluation returns one of four risk levels based on the composite score.

LOWscore 0–29

Action: allow. Clean human traffic. No friction applied.

MEDIUMscore 30–59

Action: challenge. Suspicious. Serve CAPTCHA or step-up auth.

HIGHscore 60–79

Action: throttle. Likely bot. Introduce artificial delay or rate limit.

CRITICALscore 80–100

Action: block. High-confidence bot or malicious actor. Return 429.

Signal reference

Signals are the individual detection results that contribute to the composite score. Each evaluation can return zero to many signals.

Network layer

missing_user_agentnetworkNo User-Agent header present.
bot_user_agentnetworkUA matches known bot, crawler, or automation tool pattern.
suspicious_user_agentnetworkUA is unusually short or generic.
datacenter_ipnetworkIP belongs to a cloud or hosting provider ASN.
tor_exit_nodenetworkIP is a known Tor exit node.
missing_accept_languagenetworkAccept-Language header absent — uncommon in real browsers.

Behavior layer

extreme_request_velocitybehavior3× or more above the configured request rate limit.
high_request_velocitybehavior1.5–3× above the configured request rate limit.
extreme_post_velocitybehavior3× or more above the post creation rate limit.
high_post_velocitybehaviorAbove the post creation rate limit.
machine_timing_patternbehaviorPost intervals are robotically regular (coefficient of variation < 0.05).
regular_timing_patternbehaviorPost intervals are suspiciously consistent (CV < 0.15).
fingerprint_cyclingbehaviorMore than 5 distinct browser fingerprints from one IP.

Content layer (Pro+)

crypto_wallet_addresscontentETH, BTC, or other wallet address detected in content.
high_url_densitycontentMore than 3 URLs in a single post — spam indicator.
low_content_entropycontentContent is repetitive or machine-generated (Shannon entropy < 2.5).
excessive_capscontentMore than 60% uppercase characters.
duplicate_contentcontentSame or near-identical content seen 3+ times across requests.
coordinated_duplicate_contentcontentSame content posted by 3+ distinct accounts — CIB indicator.

Account layer

brand_new_accountaccountAccount is less than 1 hour old.
new_accountaccountAccount is less than 24 hours old.
no_human_engagement_historyaccountAccount has no prior trusted interactions on record.
prior_flagsaccountAccount was previously blocked by Sentinel.

CIB layer (Pro+)

multiple_accounts_same_ipcibMore than 3 distinct accounts operating from one IP — bot farm indicator.
shared_ip_accountscib2–3 accounts sharing an IP address.
account_ip_hoppingcibOne account observed from more than 5 distinct IPs — distributed bot network.

Error codes

All errors follow a consistent JSON shape: { error, code, message }. HTTP status codes follow REST conventions.

401MISSING_API_KEYNo API key provided in Authorization or X-API-Key header.
401INVALID_API_KEYKey is invalid, revoked, or account is suspended.
400MISSING_IPThe required ip field was not provided.
400INVALID_BATCHBatch requests array is missing or empty.
400BATCH_TOO_LARGEBatch exceeds 100 items.
402MONTHLY_LIMIT_EXCEEDEDMonthly evaluation limit reached. Upgrade plan or wait for billing cycle reset.
403PLAN_FEATURE_UNAVAILABLEFeature requires a higher plan tier.
429RATE_LIMIT_EXCEEDEDPer-second rate limit exceeded. Retry after 1 second.
500EVALUATION_FAILEDInternal error. Request was not counted against your usage. Retry safely.

Webhook events

Webhooks fire within milliseconds of a block or challenge action. Payloads are signed with HMAC-SHA256 using your webhook secret.

Verifying signatures

const crypto = require('crypto');

function verifyWebhook(payload, signature, secret) {
  const expected = 'sha256=' + crypto
    .createHmac('sha256', secret)
    .update(payload)
    .digest('hex');
  return crypto.timingSafeEqual(
    Buffer.from(expected),
    Buffer.from(signature)
  );
}

// In your webhook handler:
app.post('/hooks/sentinel', express.raw({ type: 'application/json' }), (req, res) => {
  const sig = req.headers['x-sentinel-sig'];
  if (!verifyWebhook(req.body, sig, process.env.SENTINEL_WEBHOOK_SECRET)) {
    return res.status(401).send('Invalid signature');
  }
  const event = JSON.parse(req.body);
  // event.event = 'sentinel.block' | 'sentinel.challenge'
  res.json({ received: true });
});
